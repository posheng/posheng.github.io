Create Private Docker Registry (base on Nexus3)

發表於 2021-02-01 更新於 2021-02-03

Environment

MacOS 10.15.7
Docker version 20.10.2
Nexus 3.29.2

Create Private Docker Registry (base on Nexus3)
Maven Nexus3

安裝 Nexus3

1.建立一個資料夾

2.使用 docker 產生 Nexus container 並將上方產生資料夾綁定 Nexus container 中的路徑

3.連接到 Nexus web 頁面 http://YourIPAddress:8081

第一登入需要重新設定 admin 的密碼並記住該密碼
預設 admin 的密碼可以透過以下指令查到

填入新的密碼 => New password/Confrim password

勾選關閉匿名訪問 => Disable anonymous access

建立私有的 Docker Hub

1.在 Nexus 建立 Create docker-hosted 和 docker-proxy 兩個 Blob Stores

Gear(Server administration and configuration) > Repository > Blob Stores > Create blob store

Name : docker-hosted
Name : docker-hub

2.在 Nexus 建立 Create docker-hosted 和 docker-proxy 兩個 Repository

Gear(Server administration and configuration) > Repository > Repositories > Create repository

選擇 docker (hosted)

  • Name 輸入 docker-hosted
  • 勾選 HTTP 並輸入 5000
  • 勾選 Enable Docker V1 API
  • 在 Blob store 選 docker-hosted

選擇 docker (proxy)

  • Name 輸入 docker-hub
  • 勾選 Enable Docker V1 API
  • 在 Remote storage 輸入 https://registry-1.docker.io
  • 勾選 Use Docker Hub
  • 在 Blob store 選 docker-hub

3.Set Realms on Nexus (Options)

Gear(Server administration and configuration) > Realms > Move Docker Bearer Token Realm to active > Save

4.設定 docker daemon - Set docker command http enabled (v1)(for Mac)

5.重新啟動 docker

Docker Desktop > Restart Docker

6.重新啟動 Nexus container

Push 一個 image 到 Private Docker Registry(Nexus)

1.登入 Private Docker Registry(Nexus)

2.docker pull and push 到 Private Docker Registry(Nexus)

3.檢查 image
4.新增一個 image tag
5.Push 到 Private Docker Registry(Nexus)

6.檢查 image 有沒有在 Private Docker Registry(Nexus) - 連接到 Nexus web 頁面 http://YourIPAddress:8081

Browse > docker-hosted

7.從 Private Docker Registry(Nexus) pull 剛 push 的 image

Maven package & deploy

設定 Nexus 的帳號密碼在 settings.xml

mvn deploy:deploy-file

修改 version(0.0.1-SNAPSHOT or 0.0.1) 並使用 deploy-file 的方式執行 mvn deploy

mvn deploy

1.新增 distributionManagement 在 pom.xml
2.修改 version(0.0.1-SNAPSHOT or 0.0.1) 

若是 0.0.1-SNAPSHOT 會 deploy 到 maven-snapshots 否則就 deploy 到 maven-releases
