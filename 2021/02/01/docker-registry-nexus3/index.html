Create Private Maven\Docker Registry (base on Nexus3)

發表於 2021-02-01 更新於 2021-02-05

Environment

MacOS 10.15.7
Docker version 20.10.2
Nexus 3.29.2

Create Private Docker Registry (base on Nexus3)
Maven Nexus3
安裝 Nexus Repository Manager 作為 Docker Hub Mirror & Docker Image Proxy
Nexus 3 Docker倉庫（hosted、proxy、group）模式 講解

安裝 Nexus3

1.建立一個資料夾

2.使用 docker 產生 Nexus container 並將上方產生資料夾綁定 Nexus container 中的路徑

3.連接到 Nexus web 頁面 http://YourIPAddress:8081

  • 預設 admin 的密碼可以透過以下指令查到 第一次登入需要重新設定 admin 的密碼，並且記住修改的密碼

填入新的密碼 => New password/Confrim password

勾選關閉匿名訪問 => Disable anonymous access

Repository 類型

hosted : 本地存儲。像官方倉庫一樣提供本地私庫功能
proxy : 提供代理其它倉庫的類型
group : 組類型，能夠組合多個倉庫爲一個地址提供服務

建立 hosted(priavete) Repository

1.建立 Blob Stores

Gear(Server administration and configuration) > Repository > Blob Stores > Create blob store

  • Name : 輸入 docker-store-private
  • Path : 會自動產生。預設會在 ~/Dockers/nexus-data/blobs/
2.建立 docker(hosted) Repository

Gear(Server administration and configuration) > Repository > Repositories > Create repository

  • 選擇 docker (hosted)
  • Name : 輸入 docker-repo-private
  • Repository Connectors (option) : 勾選 HTTP 並輸入 8082
    不是必須的，若是希望直接 access 這個 Repository 可以勾選這個選項並給一個 port 然後在 docker 的 daemon.json 中設定 "insecure-registries" : ["YourIPAddress:8082", "YourIPAddress:8083"]，，重新啟動 docker 後就可以直接 docker push
  • Force basic authentication (option) : 勾選後就無法匿名存取了，執行 docker pull 和 docker push 都需要先登入(docker login)
  • Docker Registry API Support : 勾選 Enable Docker V1 API (為了兼容性)
  • Storage
    • Blob store : 選 docker-store-private(Blob Stores)
  • Delpoyment policy : 選擇 Allow redeploy

建立 proxy Repository

1.建立 Blob Stores

Gear(Server administration and configuration) > Repository > Blob Stores > Create blob store

  • Name : 輸入 docker-store-proxy
  • Path : 會自動產生。預設會在 ~/Dockers/nexus-data/blobs/
2.建立 docker(proxy) Repository

Gear(Server administration and configuration) > Repository > Repositories > Create repository

  • 選擇 docker (proxy)
  • Name : 輸入 docker-repo-proxy
  • proxy
    • Remote Storage : 填寫 https://registry-1.docker.io
    • Docker Index : 勾選 Use Docker Hub
  • Storage
    • Blob store : 選 docker-store-proxy(Blob Stores)

建立 group Repository

1.建立 Blob Stores

Gear(Server administration and configuration) > Repository > Blob Stores > Create blob store

  • Name : 輸入 docker-store-group
  • Path : 會自動產生。預設會在 ~/Dockers/nexus-data/blobs/
2.建立 docker(group) Repository
  • 選擇 docker (group)
  • Name : 輸入 docker-repo-group
  • 勾選 HTTP 輸入 8083 用來啟用 8083 port 的 http 連接器
  • Storage
    • Blob store : 選 docker-store-group(Blob Stores)
  • Group : 將左邊可選的 Repository 新增到右邊的 Members 下

Set Realms on Nexus (Options)

Gear(Server administration and configuration) > Realms > Move Docker Bearer Token Realm to active > Save

設定 docker daemon

重新啟動 docker

Docker Desktop > Restart Docker

重新啟動 Nexus container

Docker login(hosted 和 group)

由 Remote pull 一個 image

會有優先由 Hosted(Private) Repository 查找，沒有的話會再從 Docker Proxy Repository(https://registry-1.docker.io) 查找

檢查 image
新增一個 image tag 並且 tag 一個 image
Push 到 Hosted(Private) Repository

檢查 image 有沒有在 Hosted(Private) Repository - 連接到 Nexus web 頁面 http://YourIPAddress:8081

Browse > docker-repo-private

從 Private Docker Registry(Nexus) pull 剛 push 的 image

相關指令

Maven package & deploy

設定 Nexus 的帳號密碼在 settings.xml

mvn deploy:deploy-file

修改 version(0.0.1-SNAPSHOT or 0.0.1) 並使用 deploy-file 的方式執行 mvn deploy

mvn deploy

1.新增 distributionManagement 在 pom.xml
2.修改 version(0.0.1-SNAPSHOT or 0.0.1) 

若是 0.0.1-SNAPSHOT 會 deploy 到 maven-snapshots 否則就 deploy 到 maven-releases
